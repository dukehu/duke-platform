# 文件服务设计文档

# 一、服务整体定位

1\. 独立文件微服务，**只管文件，不管业务**。

2\. 所有文件MD5**全部后端计算**，前端不参与算MD5，安全防篡改。

3\. 能力包含：普通上传、后端MD5秒传、大文件断点分片续传、下载、预览、逻辑删除、分页查询。

4\. 存储支持：本地磁盘存储 / MinIO对象存储，配置一键切换，业务代码不动。

5\. 自带定时任务：自动清理废弃分片、自动清理逻辑删除过期文件，防止磁盘爆满。

6\. 存储自动按年/月/日分文件夹归档，不堆目录，不卡顿。

# 二、技术栈固定要求

\- Java 21 \+ SpringBoot 3\.x

\- MyBatis\-Plus

\- 本地文件IO \+ MinIO SDK（可选）

\- 后端原生文件流计算MD5工具类

\- Spring Task 定时任务，不用第三方框架

# 三、核心核心规则（必须严格遵守）

✅ **所有文件MD5，一律后端接收文件流之后计算，前端不准传任何MD5过来**

✅ 秒传：后端算出完整文件MD5，查表存在就直接返回文件信息，不用重复上传

✅ 分片只做断点续传，分片本身不用MD5，只靠chunkId\+分片序号区分

✅ 分片全部合并完成后，后端再统一算一次完整文件MD5，落库做主键去重

✅ 文件服务和知识库完全解耦，知识库单独建关联表拿fileId绑定，本表不动

# 六、完整接口清单（后端算MD5版）

后端接收文件 → 存盘 → 后端算MD5 → 查重秒传 → 落sys\_file

2\. 校验秒传（后端内部算，前端只传文件名和大小）：GET /check/exist

3\. 文件分页列表：GET /list

4\. 文件详情：GET /info/{fileId}

5\. 文件下载：GET /download/{fileId}

6\. 文件在线预览：GET /preview/{fileId}

7\. 逻辑删除文件：DELETE /{fileId}

8\. 断点查询已传分片：GET /chunk/check

9\. 分片上传：POST /chunk/upload

10\. 分片合并：POST /chunk/merge
合并完自动后端算完整MD5 → 写入主表 → 清分片数据

# 九、核心业务流程（后端MD5版）

1\. 前端上传文件流到后端接口

2\. 后端校验后缀、大小、安全校验

3\. 后端一边读取文件流，一边计算**完整文件MD5**

4\. 查表sys\_file有没有这个MD5

5\. 有：直接秒传，返回已有文件信息，不重复存文件

6\. 没有：正常保存文件、按日期分目录、写入sys\_file

7\. 大文件自动走断点分片，分片不用MD5，合并后再算MD5

8\. 定时任务自动打扫垃圾，全程不用人工管
