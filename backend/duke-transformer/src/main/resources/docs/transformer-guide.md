# Transformer 完整学习指南

> 从零开始理解 Transformer 架构，特别是 QKV 注意力机制的工作原理。

---

## 1. Transformer 是什么？

### 1.1 历史背景

在 Transformer 出现之前，序列建模主要依赖 **RNN（递归神经网络）**：

```
输入序列：[token_1, token_2, token_3, ...]
         ↓ 顺序处理
RNN:    h_1 → h_2 → h_3 → ...
         ↑ 依赖前一时刻的隐状态
```

**RNN 的问题**：
- **顺序依赖**：必须一个接一个地处理，无法并行
- **长距离衰减**：当序列很长时，早期的信息对后面的预测影响减弱（梯度消失）
- **计算效率低**：无法充分利用现代 GPU 的并行能力

### 1.2 Transformer 的革命

2017 年论文《Attention Is All You Need》提出了 Transformer，完全改变了这一局面：

```
核心思想：用"注意力机制"代替循环结构

优势：
✓ 支持并行处理整个序列
✓ 长距离依赖建模能力强
✓ 可扩展性好（容易堆叠深层网络）
✓ 成为现代 LLM 的基础（GPT、BERT、LLaMA 等）
```

### 1.3 一句话总结

**Transformer = 自注意力机制 + 前馈网络 + 层归一化 + 位置编码**

## 2. 注意力机制的直觉理解

### 2.1 生活中的类比

想象你在阅读这句话：

> "银行拒绝了贷款申请，因为**它**太危险了。"

你需要理解"**它**"指的是什么。你的大脑会：
1. 看到"它"（Query）
2. 回头扫描整个句子（Key）
3. 特别关注"贷款申请"（高相关性）
4. 从"贷款申请"的内容（Value）理解"它"的含义

这就是注意力的核心思想。

### 2.2 数学直觉

注意力机制可以理解为一个**加权查询过程**：

```
输入：句子中的所有词语
     ↓
Query（查询）：我要找什么？
Key（键）：   每个词语能匹配什么？
Value（值）：每个词语的实际内容是什么？
     ↓
相关度打分：Query 和 Key 的匹配程度
     ↓
加权求和：用高相关度的 Value 加权组合
     ↓
输出：携带上下文信息的新表示
```

### 2.3 与全连接层的区别

**全连接层**：输出的每个位置都依赖于所有输入位置，权重是固定的。

**注意力层**：输出的每个位置都依赖于所有输入位置，**但权重是根据内容动态计算的**。

这种动态性使注意力机制更灵活、更强大。

## 3. QKV 详解（核心概念）

### 3.1 Q、K、V 是什么？

自注意力机制的三个核心概念：

| 缩写 | 全称 | 中文 | 含义 | 来源 |
|------|------|------|------|------|
| Q | Query | 查询 | 当前位置想要寻找什么信息 | 同一输入 × W_Q |
| K | Key | 键 | 每个位置能提供的索引或标签 | 同一输入 × W_K |
| V | Value | 值 | 每个位置实际携带的信息内容 | 同一输入 × W_V |

**关键理解**：Q、K、V **都来自于同一个输入**，只是通过不同的权重矩阵投影。

### 3.2 具体计算流程

#### 步骤 1：输入和嵌入

```
输入文本：  "hello"
Token ID:   [7, 4, 11, 11, 14]

Embedding（查表）：
X ∈ ℝ^(5 × 16)     （5 个 token，每个 16 维）

    d0      d1      d2   ...   d15
h  [ 0.0523  -0.0341   0.0812 ...  0.0190]
e  [-0.0271   0.0634  -0.0123 ...  0.0445]
l  [ 0.0451   0.0122   0.0234 ...  0.0156]
l  [ 0.0451   0.0122   0.0234 ...  0.0156]
o  [ 0.0152  -0.0409   0.0567 ...  0.0289]
```

#### 步骤 2：线性投影得到 Q、K、V

权重矩阵（可学习）：

```
W_Q ∈ ℝ^(16 × 16)   （从 embedding 维度投影到自己）
W_K ∈ ℝ^(16 × 16)
W_V ∈ ℝ^(16 × 16)
```

计算：

```
Q = X × W_Q    ∈ ℝ^(5 × 16)
K = X × W_K    ∈ ℝ^(5 × 16)
V = X × W_V    ∈ ℝ^(5 × 16)
```

**Q、K、V 的形状完全相同**，但内容不同（因为使用了不同的权重矩阵）。

#### 步骤 3：计算相关度（点积）

为了知道每个位置与其他位置的关系，我们计算 **Query 和 Key 的点积**：

```
scores = Q × K^T    ∈ ℝ^(5 × 5)

Q: 5×16      K^T: 16×5      scores: 5×5
```

**scores[i][j] 的含义**：
- i = 查询位置（哪个 token 在提问）
- j = 键位置（哪个 token 在被查询）
- 值 = 两者的相似度（高分 = 高相关性）

**具体例子**：

```
        h      e      l      l      o
h  [ 0.312  -0.128   0.445   0.445   0.201]
e  [ 0.245   0.523   0.112   0.112   0.389]
l  [ 0.401   0.267   0.678   0.678   0.234]
l  [ 0.401   0.267   0.678   0.678   0.234]
o  [ 0.156   0.312   0.198   0.198   0.567]
```

解读：
- scores[0][3] = 0.445：token 'h' 对第二个 'l' 的相关度是 0.445
- scores[3][3] = 0.678：token 'l' 对自己的相关度是 0.678（通常最高）
- scores[0][1] = -0.128：token 'h' 对 'e' 的相关度是负数（不相关）

#### 步骤 4：缩放（防止梯度消失）

点积的结果可能很大（通常随着维度增加而增加）。如果值太大：
- softmax 时，最大值会压倒其他值（大的导数 ≈ 0）
- 梯度消失，模型难以训练

**解决方案**：除以 √d_k

```
d_k = embedding_dim / num_heads = 16 / 2 = 8

scaled_scores = scores / √8 ≈ scores / 2.828
```

这样做：
- 保持数值在合理范围内
- softmax 的输出更平滑，梯度更稳定

#### 步骤 5：Softmax（转换为概率）

用 Softmax 将得分转换为**概率分布**，每行的和为 1.0：

```
attention_weights = softmax(scaled_scores)    ∈ ℝ^(5 × 5)
```

Softmax 公式（以第一行为例）：

```
e^x = [e^0.11, e^-0.045, e^0.157, e^0.157, e^0.071]
    = [1.116, 0.956, 1.170, 1.170, 1.074]

sum_e^x = 5.486

attention_weights[0] = [1.116/5.486, 0.956/5.486, 1.170/5.486, 1.170/5.486, 1.074/5.486]
                     = [0.203, 0.174, 0.213, 0.213, 0.197]
```

**含义**：位置 0（token 'h'）对其他位置的关注权重：
- 对位置 0 自己：20.3%
- 对位置 1（'e'）：17.4%
- 对位置 2（第一个 'l'）：21.3%
- 对位置 3（第二个 'l'）：21.3%
- 对位置 4（'o'）：19.7%

#### 步骤 6：加权求和（提取信息）

最后，用注意力权重对 **Value 加权求和**：

```
output = attention_weights × V    ∈ ℝ^(5 × 16)

output[0] = 0.203 × V[0] + 0.174 × V[1] + 0.213 × V[2] + 0.213 × V[3] + 0.197 × V[4]
```

**output[0] 的含义**：
- 这是位置 0（token 'h'）经过注意力融合后的新表示
- 它融合了所有 5 个位置的信息，权重由 h 与其他 token 的相似度决定
- 权重越高的 token，对 h 的新表示的影响越大

### 3.3 完整公式

自注意力机制可以用一个简洁的数学公式表示：

$$\text{Attention}(Q, K, V) = \text{softmax}\left(\frac{Q K^T}{\sqrt{d_k}}\right) V$$

### 3.4 为什么这样设计有效？

| 组件 | 作用 | 为什么重要 |
|------|------|----------|
| Q | 表示"当前位置要找什么" | 使每个位置有不同的关注焦点 |
| K | 表示"每个位置能提供什么" | 让相似的位置相互关注 |
| V | 表示"实际的信息内容" | 确保最后融合的是有意义的内容 |
| 点积 | 衡量相似度 | 简单高效，接近全量搜索 |
| softmax | 转换为概率分布 | 保证权重有语义意义，且可微 |
| √d_k | 缩放 | 保持数值稳定，梯度有意义 |

## 4. 多头注意力（Multi-Head Attention）

### 4.1 为什么需要多个头？

单个注意力头虽然强大，但**只能学到一种关系模式**。

比如在句子"银行拒绝了贷款，因为它太危险"中：
- Head 1 可能学到：代词指代关系（"它" → "贷款"）
- Head 2 可能学到：动词宾语关系（"拒绝" → "贷款"）
- Head 3 可能学到：形容词修饰关系（"危险" → 描述对象）

多个头**从不同的语义角度**来理解句子，最后的输出是这些观点的综合。

### 4.2 多头的分解机制

```
embedding_dim = 16
num_heads = 2
d_k = embedding_dim / num_heads = 8

Q [5×16] → split → Q_head_0 [5×8]
                 → Q_head_1 [5×8]

K [5×16] → split → K_head_0 [5×8]
                 → K_head_1 [5×8]

V [5×16] → split → V_head_0 [5×8]
                 → V_head_1 [5×8]
```

### 4.3 并行计算

```
head_0: Attention(Q_head_0, K_head_0, V_head_0) → output_0 [5×8]
head_1: Attention(Q_head_1, K_head_1, V_head_1) → output_1 [5×8]

【独立计算，可并行执行】

concat(output_0, output_1) → [5×16]
                    ↓
            × W_O [16×16]  （输出投影）
                    ↓
            Multi-Head Output [5×16]
```

### 4.4 为什么最后要拼接？

拼接后的矩阵 [5×16] 回到原始维度，可以继续通过网络的其他部分（如 FFN）处理。

通过 W_O 的线性变换，模型学会了**如何综合不同头的信息**。

## 5. 位置编码（Positional Encoding）

### 5.1 问题：Transformer 无序列感知

考虑两个不同的句子：
- "爱丽丝喜欢鲍勃"
- "鲍勃喜欢爱丽丝"

在 Transformer 中，这两个句子的 token 顺序不同，但如果我们**只考虑自注意力**，不加特殊处理，模型可能会混淆两者的意义。

原因：自注意力对所有位置一视同仁，没有"位置"的概念。

### 5.2 解决方案：正弦位置编码

论文采用了一种巧妙的方案：**不学习位置编码，而是用数学公式生成**。

```
PE(pos, 2i) = sin(pos / 10000^(2i / d_model))
PE(pos, 2i+1) = cos(pos / 10000^(2i / d_model))
```

其中：
- pos：位置索引（0, 1, 2, ...）
- i：维度索引（0, 1, ..., d_model/2-1）
- d_model：嵌入维度（这里是 16）

### 5.3 具体例子

对于 d_model = 16，pos = 0（第一个 token）：

```
PE[0] = [sin(0), cos(0), sin(0), cos(0), sin(0), cos(0), ...]
      = [0, 1, 0, 1, 0, 1, ...]

对于 d_model = 16，pos = 1（第二个 token）：

PE[1] = [sin(1/10000^0), cos(1/10000^0), sin(1/10000^(2/16)), cos(1/10000^(2/16)), ...]
      ≈ [0.841, 0.540, 0.839, 0.544, ...]
```

### 5.4 特性和优势

| 特性 | 优势 |
|------|------|
| 使用 sin/cos | 周期性，能编码任意位置 |
| 不同频率 | 低频维度编码粗粒度位置，高频维度编码细粒度位置 |
| 可外推 | 即使测试时出现比训练更长的序列，也能自动编码 |
| 不可学习 | 减少参数，加快训练 |

### 5.5 可视化

```
Position 0: [0.0, 1.0, 0.0, 1.0, 0.0, 1.0, ...]
Position 1: [0.8, 0.5, 0.8, 0.5, 0.8, 0.5, ...]
Position 2: [0.9, -0.1, 0.9, -0.1, 0.9, -0.1, ...]
...

每个位置都有独特的"位置指纹"，模型可以学到位置与语义的关联
```

## 6. 前馈网络 + 残差连接 + 层归一化

### 6.1 前馈网络（FFN）

在自注意力之后，Transformer 有一个**前馈网络**：

```
FFN(x) = max(0, x × W1 + b1) × W2 + b2
```

**结构**：

```
[seq × d_model] 
    ↓ × W1 + b1
[seq × d_ff]        （d_ff = 2 × d_model = 32）
    ↓ ReLU（max(0, x)）
[seq × d_ff]        （应用非线性激活）
    ↓ × W2 + b2
[seq × d_model]     （投影回原维度）
```

**作用**：

- **扩展表达能力**：先扩展到高维，再压缩回来，相当于经过非线性变换
- **引入非线性**：ReLU 激活打破了线性性，增加了模型表达能力
- **位置级别的处理**：每个位置的 FFN 都是独立的，可以并行计算

### 6.2 残差连接（Residual Connection）

在深度神经网络中，**梯度消失**是一个大问题。残差连接通过**直接连接**来解决：

```
output = attention_output + input

而不是：
output = attention_output
```

**为什么有效**：

- 梯度可以通过"跳过"连接直接流向早期层
- 深层网络也能有效训练
- 当注意力层想"什么都不做"时，它可以学到输出为 0

### 6.3 层归一化（Layer Normalization）

在特征维度（每行）进行归一化：

```
LN(x) = (x - mean(x)) / sqrt(var(x) + eps)

其中：
- mean(x)：每行的平均值
- var(x)：每行的方差
- eps：小常数（防止除以 0）
```

**作用**：

- **稳定训练**：确保隐层输出在合理范围内
- **加速收敛**：更均匀的特征分布
- **减少对初始化的敏感性**：使网络更鲁棒

### 6.4 完整的 Encoder 块

```
输入：X [seq × d_model]
   ↓
Multi-Head Attention(X)
   ↓
Add：Attn_out + X
   ↓
LayerNorm
   ↓
FFN
   ↓
Add：FFN_out + Norm_out
   ↓
LayerNorm
   ↓
输出：[seq × d_model]
```

## 7. 完整 Transformer Encoder 架构

### 7.1 数据流全程

```
原始文本
   ↓
分词（Tokenization）
[token_1, token_2, ..., token_n]
   ↓
Token Embedding（查表）
X ∈ ℝ^(n × d_model)
   ↓
Position Encoding（相加）
X_pos = X + PE ∈ ℝ^(n × d_model)
   ↓
【Encoder Block × num_layers】
├─ Multi-Head Attention
├─ Add & LayerNorm
├─ FFN
├─ Add & LayerNorm
   ↓
输出表示 ∈ ℝ^(n × d_model)
   ↓
线性层 + Softmax（可选，用于分类或生成）
   ↓
最终输出
```

### 7.2 各层的维度变化

以演示参数为例（embedding_dim=16, num_heads=2）：

```
输入文本："hello"
Token IDs: [7, 4, 11, 11, 14]       维度 [5]
    ↓
Embedding: [5 × 16]
    ↓
Position: [5 × 16]
    ↓
QKV Projection:
  Q: [5 × 16]
  K: [5 × 16]
  V: [5 × 16]
    ↓
分头（2 heads）：
  Q_h0: [5 × 8]  Q_h1: [5 × 8]
  K_h0: [5 × 8]  K_h1: [5 × 8]
  V_h0: [5 × 8]  V_h1: [5 × 8]
    ↓
Attention（每个头）：
  scores_h0: [5 × 5]
  weights_h0: [5 × 5]
  output_h0: [5 × 8]
    ↓
拼接 + W_O: [5 × 16]
    ↓
FFN:
  W1: [5 × 32] (ReLU)
  W2: [5 × 16]
    ↓
最终: [5 × 16]
```

### 7.3 与 GPT、BERT 的关系

**BERT**（双向编码）：
- 使用多层 Transformer Encoder
- 用 [MASK] token 进行掩码预训练
- 输出是**双向**上下文表示

**GPT**（自回归生成）：
- 使用多层 Transformer Decoder
- 在自注意力中加 Mask（不能关注未来位置）
- 用于**文本生成**

**现代 LLM**（如 LLaMA、GPT-4）：
- 基于 Transformer Decoder
- 堆叠 50-100+ 层
- 数十亿参数规模

## 8. 总结与要点

### 8.1 核心要点

1. **Transformer 的基础是自注意力机制**
   - Q、K、V 三者协同完成"查询-匹配-提取"
   - 点积衡量相似度，softmax 转换为概率

2. **QKV 的工作流程**
   - Q = X × W_Q（"我要找什么"）
   - K = X × W_K（"你能提供什么"）
   - V = X × W_V（"你的内容是什么"）
   - Attention = softmax(Q×K^T/√d_k) × V

3. **多头注意力增强表达能力**
   - 多个头从不同角度理解序列
   - 拼接后的结果更全面

4. **位置编码保留顺序信息**
   - 正弦函数编码位置
   - 可外推到未见过的长度

5. **残差连接和层归一化稳定训练**
   - 梯度可以顺畅流动
   - 特征分布更均匀
